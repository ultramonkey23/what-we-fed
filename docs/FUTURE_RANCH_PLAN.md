# FUTURE RANCH PLAN

## Status
Deferred for later implementation.

## Why it is deferred
The ranch should not be built yet.
It will work best after:
- more creatures are defined
- creature roles are clearer
- bond/eat progression is more mature
- active bonded support is implemented and tested
- the run-growth loop is more stable

## Locked design intent
The ranch should be a between-run creature hub that:
- stores bonded creatures
- allows training and long-term development
- lets the player assign one active support creature for the next run
- reinforces the tension between bonding for long-term value and eating for immediate power
- feels dark, strange, and in-world
- does not become a giant management sim

## What the ranch should eventually do
1. Storage
   - keep bonded creatures between runs

2. Training
   - improve support efficiency
   - deepen creature role identity
   - allow slow long-term growth

3. Assignment
   - choose one active support creature for the next run

4. Tension
   - keeping creatures should have long-term value
   - eating creatures should still matter as immediate power

## What it should NOT become
- a cute pet menu
- a giant spreadsheet
- a disconnected side mode
- a system that removes the value of eating creatures

## Current priority before ranch implementation
1. timing truth / combat trust
2. creature feedback and readability
3. run-growth + bonded support
4. broader creature identity and schema
5. only then: ranch design implementation

## Revisit trigger
Return to this after:
- multiple creatures exist
- active bonded support works
- bond/eat tradeoffs are clearer in play
