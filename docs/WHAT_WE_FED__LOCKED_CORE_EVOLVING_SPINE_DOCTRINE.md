# WHAT WE FED — LOCKED CORE / EVOLVING SPINE DOCTRINE

## Purpose

This doctrine defines how WHAT WE FED should evolve going forward.

It exists to allow:
- controlled vibe coding
- creative AI drift
- abstraction
- emergent gameplay mutation
- stronger authorship
- stranger and more original evolution

without allowing the project to lose its identity.

This is not permission for random drift.
This is permission for **controlled evolution**.

---

# 1. Core Rule

**Let the game’s form mutate. Do not let its soul dissolve.**

WHAT WE FED may evolve:
- mechanics
- progression shape
- reward structure
- creature logic
- support behavior
- region identity
- boss structure
- world meaning
- presentation texture
- connective tissue between systems

But it must preserve the project’s core truth.

---

# 2. LOCKED CORE

These are the identity locks.
Future changes must not break them.

## 2.1 Gameplay truth locks
- timing truth
- lane readability
- player-side response clarity
- readable support
- combat honesty
- feedback that matches actual mechanics

## 2.2 System identity locks
- DNA as species-specific predation economy
- bond vs eat as a meaningful identity split
- realtime song-run flow
- no repeated stop-start menu interruption as the main progression spine
- behavior-shaped growth
- creatures as meaningful ecosystem units, not detached tokens

## 2.3 Fantasy / tone locks
- start weak, become feared
- dark predation
- pressure
- mutation
- consequence
- oppressive premium-feeling presentation
- authored weirdness over generic genre sludge

## 2.4 Anti-drift locks
The game must not drift into:
- generic survivor clone
- generic creature collector
- generic roguelite soup
- generic rhythm gimmick
- stat spreadsheet sludge
- stop-start draft/menu game
- pretty but unreadable action slop
- random “cool” with no gameplay truth

---

# 3. EVOLVING ZONES

These are allowed to mutate more freely if they make the game stronger.

## 3.1 Mechanics that may evolve
- level-up expression
- growth routing
- reward logic
- support proc behavior
- encounter pacing
- region pressure behavior
- boss escalation structure
- creature role expression
- score/performance payoff structure
- run divergence logic

## 3.2 Spine elements that may evolve moderately
- how bond vs eat affects progression
- how DNA routes into short-term vs long-term power
- how runs differentiate into identities
- how support ties into survivability or offense
- how performance reward depth is expressed
- how stats are owned/resolved
- how runs become “more monstrous” over time

## 3.3 Aesthetic / thematic layers that may evolve freely
- creature abstraction
- reward naming
- region mood
- world symbolism
- haunted/ecological implications
- Quig delivery style
- music lane blending
- emotional contrast
- creative connective tissue

---

# 4. CONTROLLED DRIFT RULE

AI drift is allowed when it produces one or more of these:

- stronger identity
- more authored feel
- more memorable creature/ecosystem logic
- more meaningful bond/eat tension
- more distinct run identities
- stronger emotional contradiction
- stranger but cleaner gameplay expression
- better fusion of system and tone

AI drift is NOT allowed when it produces one or more of these:

- weaker readability
- weaker timing trust
- more clutter
- softer creature meaning
- flatter bond/eat tension
- more generic progression
- decorative abstraction that replaces mechanics
- gameplay confusion masked as style
- broad mutation without real improvement

---

# 5. MUTATION TEST

When a new idea appears, test it with these questions:

## 5.1 Identity test
Does this make WHAT WE FED more itself, or less itself?

## 5.2 Gameplay truth test
Does this preserve timing honesty, lane readability, and support readability?

## 5.3 Creature meaning test
Does this strengthen creatures as enemies, DNA sources, bond/eat choices, and support identities?

## 5.4 Bond/Eat test
Does this preserve or deepen the tension between bonding and consuming?

## 5.5 No-pause test
Does this preserve the live song-run identity?

## 5.6 Practicality test
Can this be implemented without bloating the repo or muddying the player read?

## 5.7 Upgrade test
Is this a real upgrade, or just a more elaborate distraction?

If the idea fails these tests, it should be rejected or reshaped.

---

# 6. PREFERRED EVOLUTION STYLE

Future evolution should favor:

- moderate structural mutation over total redesign
- stronger hooks over more systems
- deeper roles over more filler
- live-feel improvements over menu complexity
- meaningful weirdness over generic polish
- clean surprising synergies over random novelty
- sharper run identity over broader but flatter content

The game should evolve like an organism under pressure:
- adapting
- specializing
- becoming stranger
- becoming stronger
- but still recognizably itself

---

# 7. CLAUDE / CODEX WORKFLOW RULE

When using Claude or Codex on WHAT WE FED:

## Claude should be used for:
- broader synthesis
- strategic mutation
- spine evolution
- multi-system direction
- controlled creative drift
- critique and next-step routing

## Codex should be used for:
- bounded implementation
- cleanup
- bug fixing
- local feature passes
- targeted polish
- repo-facing execution

## Shared rule
Both agents may propose mutation.
Neither agent may violate the locked core.

---

# 8. QUIG RULE UNDER THIS DOCTRINE

Quig may evolve in:
- humor style
- timing
- tone adaptation
- visual behavior
- commentary pattern

Quig must still remain:
- controlled
- useful
- readable
- tonally aware
- non-parodic
- a guide, not a derailment engine

---

# 9. PRACTICAL DECISION RULE

When choosing between:
- safer familiarity
- or controlled mutation

prefer controlled mutation **if**:
- the core remains locked
- readability remains intact
- the result is stronger, stranger, and more authored
- the gameplay truth improves or stays equally strong

Prefer stability **if**:
- the mutation muddies combat
- the identity becomes less clear
- the system becomes bloated
- the game becomes more generic or more confusing

---

# 10. ONE-SENTENCE OPERATIONAL RULE

**Preserve the locked core, but allow controlled AI-driven evolution of gameplay, structure, and style whenever it makes WHAT WE FED feel more authored, more alive, more monstrous, and more itself.**
